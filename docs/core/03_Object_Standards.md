# 03. Object Standards

## Objects
TASKFLOW-CX, CasePack-MAO, Bridge CasePack, Handoff Contract, CasePack Graph, Domain Pack Manifest, App Object, Input Contract, Output Contract, Runtime Contract, UI Schema, Validation Report, Runtime Trace Event, Usage Event.

## TASKFLOW-CX
A Role/Audience, S Situation/Scope, T Task/Target, K Knowledge Firewall, W Watchouts, F Flow, L Language/Tone, O Output Contract.

K_REF is stable reference logic. K_IN is runtime input. K_EX is examples.

## CasePack-MAO
Contains taskflow_cx, input_contract, output_contract, runtime_contract, ui_schema, policy_pack, evals, metadata.

## Bridge CasePack
Transforms source output into target input via source_pattern, target_pattern, mapping_rules, default_values, context_checkpoint, handoff_contract.

## Public Exposure
Public users may see app metadata, safe contracts, safe UI schema, trust badges, and final output. Public users must not see raw internal objects.
